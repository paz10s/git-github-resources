Terminal Commands

ls - list items in current directory

cd <path> - go to path

cd .. - go back one directory

touch <filename> - create file, only works in git bash

Basic Git Commands

git init - create a local git repo

git add <filename> - add files to staging area

git add . - add all files in current directory

git add *.html - add all files with .html extension

git reset <filename> - remove file from staging area

git commit -m "<message>" - make commit

git commit -am "<message>" - add modified files and make commit

Branches

git branch - lists branches

git branch <branchname> - create branch

git tag <tagname> - name current commit

git checkout <branchname> - check out to branch

Commits

git log - view history of commits

git reset - unstage files

git reset HEAD~1 - go back 1 commit

  • HEAD refers to the current commit

git reset <commit hash> - go back to commit corresponding to the hash

git reset --hard <commit hash> - unstage changes, and also remove all changes since that commit

.gitignore file

  • Create this through touch .gitignore
  • Type in filenames you wish for Git to ignore
  • For folders: /<foldername>

Github

Note: when creating a repo in Github.com, Github already tells you the commands you need to execute to link your remote repo!

Linking your local repo to Github

  1. Create a remote repo in github.com
  1. Use the following commands:
    • git remove add origin <link> - create a local git repo
    • git push -u origin master - add files to staging area
      • git push - add all files in current directory

Cloning a remote repo

  1. Create a remote repo in github.com
  1. Use the following command:
    • git clone <link> - clone remote repo to your system
